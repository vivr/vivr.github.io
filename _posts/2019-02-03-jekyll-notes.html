What is Jekyll?

Jekyll gives us the ability to create a static website by using dynamic content such as templates, partials and liquid code. Jekyll sites can be hosted on Github servers for free. A benefit of this type of website is that it doesn't need a database which can make the site faster and more secure. If you want to read more about Jekyll take a look at this documentation from the Jekyll website.

Project set up

This post describes how I used Jekyll to build my portfolio website. I don't give any guidance on Jekyll installation so if you want to learn how to install it take a look at the following article: Set up a Github Pages site locally with Jekyll.

  8. Create a new file called styles.css inside the assets/css
  9. Create the following folders inside the assets/img folder:
    • blog - this will contain images attached to our blog posts

  10. Im using a syntax highligher called Prism.js which consists of a single css file and a single javascript file. These files are placed in the assets/css and the assets/js folders respectively

    {% assign prevurl = posts.last.url %}
    {% assign nexturl = posts.first.url %}
  {% endif %}

  {% for links in posts %}
    {% if links.title == page.title %}

      {% endunless %}
      {% unless forloop.last %}
        {% assign next = posts[forloop.index] %}

      {% endunless %}
    {% endif %}

  {% endfor %}

Manage the _posts folder

The _posts folder contains all of our blog posts. To add a blog post first create a new file with the following format: YEAR-MONTH-DAY-title.MARKUP. For example, the React Notes blog post file name is 2018-09-12-react-notes-1.html.

All blog posts must begin with front matter. The 2018-09-12-react-notes-1.html blog post has the following front matter:


{% raw %}
---
layout: post
title: "React Notes"
categories: react
featImg: react.jpg
itemType: image
pageType: blog
date: 2018-09-12 11:22:45 +0100
---
{% endraw %}

Manage the _projects folder

The _projects folder contains all of the files in our projects collection. To add a file to the collection, create a new file inside the _projects folder. All files that are part of a collection must begin with front matter. For example, the ckw.html file has the following front matter:


{% raw %}
---
layout: post
title: CKW Trading
featImg: ckw.jpg
itemType: image
pageType: project
date: 2018-08-06 11:17:59 +0100
---
{% endraw %}
